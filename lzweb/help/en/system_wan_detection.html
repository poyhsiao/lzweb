WAN Detection

This page allows you to configure detection settings for each WAN link. XteraLink uses both ICMP and TCP to detect WAN link, and determines each WAN link's health based on received packets.

Always Success When There Is Inbound Traffic When this option is disabled (unchecked), XteraLink will not determine the status of WAN connection based on inbound traffic.
When enabled (checked), XteraLink will determine this WAN link as active as long as inbound traffic is detected, without further detecting ICMP and TCP packets.

WAN Select a WAN link to be configured.

Protocol Select a WAN link detection method: ICMP or TCP.
Detection Period This indicates the detection interval in seconds. A shorter interval means faster detection, but takes bigger bandwidth.
Number of Targets per Detection For each detection, system will randomly sort out a number of IP addresses from the target list for detection. The number of selected IP addresses shall not exceed the total number in target list.
Number of Retries This indicates the number of detection retries when error occurs. If all retries fail, XteraLink will determine the WAN link unavailable.

Target List When ICMP protocol is used
Destination IPThe IP address of the host where "ping" is used for detection.
HopsThe time to live of "ping" command is defined by the parameter Hops which is usually 3.
When TCP protocol is used
Destination IPThe IP address of the host where "TCP connect" is used for detection
TCP PortThe assigned port number
