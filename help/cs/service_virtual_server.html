PPPoe：使用该广域网络利用PPPoe分配到的第一个IP地址

服务
任何服务虚拟服务器使用任何服务项目
协议@虚拟服务器使用协议号码分类的服务封包，例如：协定@1 为 ICMP 封包
TCP@虚拟服务器使用以TCP通讯端口号码分类的服务封包，例如：TCP@80 为 HTTP 封包
UDP@虚拟服务器使用以UDP通讯端口号码分类的服务封包，例如：UDP@53 为 DNS 封包
群组@虚拟服务器使用符合 系统/网络服务群组设定 分页中规则的封包，例如：群组@号码(名称)
真实服务器

端口对应设定实际主机的对应通讯端口
日志
不打勾(Disable)当此条规则有被引用到时不会加入 日志/记录浏览/流量 分页中
打勾(Enable)当此条规则有被引用到时会加入 日志/记录浏览/流量 分页中

注意：
(1)当 虚拟主机/服务 设定为 任何服务、协议@、群组@ 时，使用者无法设定 真实服务器/端口对应，即虚拟主机对应相同的服务至真实服务器

(2)当 虚拟主机/服务 设定为 TCP@ 时，使用者必须设定 真实服务器/端口对应，例如：
[单一端口对应] 虚拟主机/服务 为 TCP@80，真实服务器/端口对应 应为 TCP@80 (TCP＠22对应到TCP@80)
[单一端口转换] 虚拟主机/服务 为 TCP@22，真实服务器/端口对应 应为 TCP@23 (TCP@22转换到TCP@23)
[多端口对应] 虚拟主机/服务 为 TCP@22-25，真实服务器/端口对应 应为 TCP@22-25 (TCP＠22对应到TCP@22，TCP＠23对应到TCP@23，以此类推)
[多端口转换] 虚拟主机/服务 为 TCP@22-25，真实服务器/端口对应 应为 TCP@32-35 (TCP＠22转换到TCP@32，TCP＠23转换到TCP@33，以此类推)

(3)当 虚拟主机/服务 设定为 UDP@ 时，使用者必须设定 真实服务器/端口对应，例如：
[单一端口对应] 虚拟主机/服务 为 UDP@80，真实服务器/端口对应 应为 UDP@80 (UDP＠22对应到UDP@80)
[单一端口转换] 虚拟主机/服务 为 UDP@22，真实服务器/端口对应 应为 UDP@23 (UDP@22转换到UDP@23)
[多端口对应] 虚拟主机/服务 为 UDP@22-25，真实服务器/端口对应 应为 UDP@22-25 (UDP＠22对应到UDP@22，UDP＠23对应到UDP@23，以此类推)
[多端口转换] 虚拟主机/服务 为 UDP@22-25，真实服务器/端口对应 应为 UDP@32-35 (UDP＠22转换到UDP@32，UDP＠23转换到UDP@33，以此类推)

(4) TCP与UDP无法做端口对应或转换，必须是TCP对TCP或UDP对UDP

(5) 多端口对应或转换时，虚拟主机/服务 与 真实服务器/端口对应 必须是相同的端口数量
