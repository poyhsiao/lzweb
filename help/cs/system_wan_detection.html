广域网络侦测

在这个页面可以针对各广域网络联机的侦测方式做设定，XteraLink本身使用ICMP和TCP封包对网络的联机状态做侦测，依照回传的状况决定此广域网络联机是否正常。

当有对内流量,总是成功 当不启用(Disable)此设定时，XteraLink就不会利用对内的广域网络流量来判断线路联机状况
当启用(Enable)此设定时，只要XteraLink有侦测到 对内的广域网络流量，就判定此路广域网络联机是属于正常联机状态，而不再进行icmp和tcp封包的侦测行为

广域网络 选择欲设定的广域网络联机

协定 可选择要进行广域网络联机状态侦测的方式，可分成ICMP和TCP两种
侦测周期 表示每次侦测的时间间隔，以秒表示，间隔短表示可以更快发现目前的联机状况，不过也会消耗比较多的带宽
每次侦测所选出之目标数量 在每次侦测的时候，随机从目标清单选出不同的IP地址的数量做侦测，不可高于目标清单的数量
重试次数 发生错误时的重测次数，重复上述的测试多次，如果所有的测试都失败，XteraLink则宣告此广域网络联机无法运作

目标清单 协议为ICMP封包状态下
目标 IP 地址利用Ping来进行网络侦测的主机地址
跃点这个ping的存活时间 (time to live) 是由参数「跃点」 (Hops)所定义，一般设定为3
协议为TCP封包状态下
目标 IP 地址利用TCP connect来进行网络侦测的主机地址
TCP 端口指定「通讯端口」 (Port)的数值
