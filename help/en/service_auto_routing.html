Auto Routing

Auto Routing SettingThis table allows you to configure Method, Parameters, and Persistence Time for each rule.
MethodSelect whether auto routing should be based on weight or traffic. e.g.
Weight: Auto Routing is determined by user-defined parameters.
Traffic: XteraLink will establish new connection on low-traffic WAN link based on current traffic flow.
ParametersIf Weight is selected as the method, please enter the weight ratio of each WAN link. e.g.
WAN1:WAN2:WAN3
1:1:1
Persistence TimeEnter a duration time for persistent routing.

IPv4 RulesThis table allows you to configure Source, Destination, Service, Failover, Persistent and Log for each rule.
Source
IPv4 AddressPackets from a specified IP address will be matched. e.g. 192.168.1.4
IPv4 Range Packets from certain hosts within a specified IP range will be matched. e.g. 192.168.1.10-192.168.1.20
Subnet/maskPackets from a specified subnet will be matched. e.g. 192.168.1.0/255.255.255.0
LANAny packet from LAN will be matched.
DMZAny packet from DMZ will be matched.
WANAny packet from WAN will be matched.
AnyPackets from any IP address will be matched.
group@Packets from a specified group (defined in [System/IP Group]) will be matched. e.g. group@number(name)
FQDN@Packets from a specified FQDN number (defined in [System/FQDN]) will be matched. e.g. FQDN@number(domain name)
Destination
IPv4 AddressPackets to a specified IP address will be matched. e.g. 192.168.1.4
IPv4 RangePackets to certain hosts within a specified IP range will be matched. e.g. 192.168.1.10-192.168.1.20
Subnet/maskPackets to a specified subnet will be matched. e.g. 192.168.1.0/255.255.255.0
LANAny packet to LAN will be matched.
DMZAny packet to DMZ will be matched.
WANAny packet to WAN will be matched.
LocalhostAny packet to the XteraLink will be matched.
AnyPackets to any IP address will be matched.
group@Packets to a specified group (defined in [System/IP Group]) will be matched. e.g. group@number(name)
FQDN@Packets to a specified FQDN number (defined in [System/FQDN]) will be matched. e.g. FQDN@number(domain name)
Service
AnyNo service will be matched.
proto@Packets of services categorized by protocol number will be matched. e.g. proto@1 indicates ICMP packets.
TCP@Packets of services categorized by TCP port will be matched. e.g. TCP@80 indicates HTTP packets.
UDP@Packets of services categorized by UDP port will be matched. e.g. UDP@53 indicates DNS packets.
group@Packets of service group defined in [System/Service Group] will be matched. e.g. group@number(name)
WAN
CheckSelect the WAN link that applies Auto Routing.
Select one WAN: All connections that matches the IPv4 rule will be routed to this selected WAN.
Select more than one WAN: Connections that match the IPv4 rule will be auto routed to one of the selected WAN links based on selected method.
Failover
Checked (Enabled)When failover is enabled, system will attempt to match connections with first IPv4 rule, and continue on with the next one if first attempt fails, and so on.
Unchecked (Disabled)When failover is disabled, system will not attempt to match connections with next IPv4 rule if first attempt fails.
Persistent
Checked (Enabled)When connections match this rule, auto routing will be persistent on selected WAN link(s) based on the persistent time defined.
Unchecked (Disabled)Persistent routing is not applied to selected WAN link(s) even when connections match this rule.
Log
Checked (Enabled)The record of this rule being used will not be logged to the page of [Log/View/Traffic].
Unchecked (Disabled)The record of this rule being used will be logged to the page of [Log/View/Traffic].
