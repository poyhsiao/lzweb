Connection Limit

IPv4 Rule This table allows you to configure the Source, Destination, Service, Connection/Sec and Log for each IPv4 rule.
Source
IPv4 AddressEnter an IP address where connections are established from. e.g. 192.168.1.4
IPv4 RangeEnter an IP range of hosts where connections are established from. e.g. 192.168.1.10-192.168.1.20
Subnet/maskEnter a subnet address where connections are established from. e.g. 192.168.1.0/255.255.255.0
LANSelect LAN as the source where connections are established from.
DMZSelect DMZ as the source where connections are established from.
WANSelect WAN as the source where connections are established from.
AnySelect any source where connections are established from.
group@Select a group defined in [System/IP Group] as the source where connections are established from. e.g. group@number(name)
FQDN@Select an FQDN number defined in [System/FQDN] as the source where connections are established from. e.g. FQDN@number(domain name)
Destination
IPv4 AddressEnter an IP address where connections are established to. e.g. 192.168.1.4
IPv4 RangeEnter an IP range of hosts where connections are established to. e.g. 192.168.1.10-192.168.1.20
Subnet/maskEnter a subnet address where connections are established to. e.g. 192.168.1.0/255.255.255.0
LANSelect LAN as the destination where connections are established to.
DMZSelect DMZ as the destination where connections are established to.
WANSelect WAN as the destination where connections are established to.
LocalhostSelect localhost as the destination where connections are established to.
AnySelect any destination where connections are established to.
group@Select a group defined in [System/IP Group] as the destination where connections are established to. e.g. group@number(name)
FQDN@Select an FQDN number defined in [System/FQDN] as the destination where connections are established to. e.g. FQDN@number(domain name)
Service
AnyNo service will be matched.
proto@Enter a protocol number categorized for specific service to be matched. e.g. proto@1 indicates ICMP packets.
TCP@Enter a TCP number categorized for specific service. e.g. TCP@80 indicates HTTP packets.
UDP@Enter a UDP number categorized for specific service. e.g. UDP@53 indicates DNS packets.
group@Enter a group number (defined in [System/IP Group]) categorized for specific service. e.g. group@number(name)
Conn/SecThis indicates the connection number allowed to pass through XteraLink per second.
Connections that exceed the number entered will be aborted by XteraLink.
Log
Unchecked (Disable)The record of connection number per second that exceeds configured value will not be logged in the page of [Log/View].
Checked (Enable)The record of connection number per second that exceeds configured value will be logged in the page of [Log/View] once every 5 minutes.
